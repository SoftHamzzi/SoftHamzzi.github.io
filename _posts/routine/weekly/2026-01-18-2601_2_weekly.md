---

### 4번 말했는데 0번 실행

**"GPT = 보조 수단" = 5번째 원칙 수립**

하지만:
- 1/15 → 1/17 = GPT 의존 지속
- 1/17 → 1/18 = GPT 의존 지속
- 1/18 → 주간 = GPT 의존 지속

**원칙 5번 수립 = 실행 0번 = 문제는 원칙이 아니라 실행 방법입니다.**

---

### 왜 실행이 안 되나?

**"보조 수단" = 추상적이기 때문입니다.**

- 어디까지가 보조? (구조 설계? 함수 작성? 디버깅?)
- 주도적 개발 = 어떻게? (GPT 코드 안 보기? 수정하기?)
- 실행 여부 = 누가 확인? (자기 점검? 기록?)

**추상적 = 실행 어려움 = 5번 반복**

---

### 해결: 구체적 강제 장치

내일(1/20) 월요일부터 **즉시 실행**:

#### 1. GPT 사용 = 하루 3회 제한
- **종이에 체크** (GPT 질문할 때마다 正 표시)
- 3회 초과 = 금지
- 일간 회고 = "오늘 GPT 2회 사용"

#### 2. GPT 코드 받으면 = 30분 분석 의무
- **타이머 30분 설정**
- 각 줄 **주석 추가** (무슨 역할?)
- 이해 안 되면 GPT "설명만" 요청 (코드 추가 요청 X)
- 일간 회고 = "30분 분석 완료 ✅"

#### 3. 새 기능 = GPT 없이 1시간 시도
- **타이머 1시간**
- 막히는 부분 **메모**
- 1시간 후 → GPT 질문 (1회 카운트)
- 일간 회고 = "1시간 직접 시도 완료 ✅"

#### 4. 일간 회고 = 실행 여부 기록
```
- GPT 사용: 2회 (구조 설계, 디버깅)
- 30분 분석: 완료 ✅
- 1시간 직접 시도: 완료 ✅
```

**GGJ = 11일 후 = 마지막 기회**

GGJ = 48시간 = 팀원과 협업 = GPT 의존 불가능:
- 버그 나면 → 직접 고쳐야 함
- 팀원 질문 → 직접 설명해야 함
- 새 기능 → 직접 구현해야 함

**지금 GPT 의존 = GGJ에서:**
- 코드 이해 못 함 = 버그 못 고침
- 설명 못 함 = 팀원 실망

**"실력 안 늠" = 정확한 진단입니다. 지금 고치지 않으면 = GGJ 망칩니다.**
